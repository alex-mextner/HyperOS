# Naming Раунд 3 (10.07.2026): базис, кость, Рим

Вводные: fluid уходит не в ту коннотацию (отбросить FluidSpace/Fluidum как имена ОС — остаются внутренними терминами). Новые направления: антихрупкость, адаптивность, костная ткань, децентрализация, local-first, античность. Цель — найти квинтэссенцию: минимальный базис, из которого выводится большинство идей системы, и назвать ОС от него.

---

## Часть I. Квинтэссенция: минимальный базис

Прогнал все идеи из «Своя мобильная ОС» и UX RESEARCHES на предмет «что из чего выводится». Получилось три несократимых аксиомы. Всё остальное — следствия.

### Аксиома 1. Всё есть сущность-в-графе (не в приложении)
Person, task, document, event, message, device — узлы; связи типизированы, со временем и источником.
**Выводится:** entity-first · TagFS · глобальная история (граф во времени) · SideMemo (рёбра из текста) · FluidSpace (один контент — много представлений) · person-first мессенджер · User as API · миграция схем.

### Аксиома 2. Данные и логика живут локально; сеть — это синк, не владелец
Local-first + CRDT + 100% бекап состояния + capability-доступ.
**Выводится:** приватность · офлайн · 100% бекап включая память · decentralized-синк · Keyhive-доступ · «данные у человека, вендоры просят» · антивендор-лок.

### Аксиома 3. Всё, что фиксируется в графе, доступно агентам и композиции
100% покрытие действий/данных сквозным слоем; агенты и модули поверх; всё компонуется.
**Выводится:** agentic-слой · IntentBox/FlexLight · Proactive UI · микро-приложения · malleability · MCP наружу · «coverage by construction vs opt-in».

**Квинтэссенция одной фразой:**
> **Локальный типизированный граф всего, открытый агентам.**
> (local-first typed graph of everything, open to agents)

Три ключевых слова базиса: **граф / кость (несущая структура) / местное (local)**. Имя в идеале указывает на один из них — лучше на несущую структуру, потому что она объединяет «граф» (топология) и «антихрупкость» (перестраивается под нагрузкой).

### Почему «костная ткань» — точная метафора (а не просто красивая)
Трабекулярная (губчатая) кость — это буквально граф несущих балок, который **перестраивается под нагрузкой** по закону Вольфа: где нагрузка — там нарастает материал, где нет — рассасывается. Это одновременно:
- **граф** (сеть перемычек-трабекул) → Аксиома 1;
- **местная перестройка** (остеокласт/остеобласт работают локально, без центрального плана) → Аксиома 2, децентрализация;
- **антихрупкость по Талебу** (нагрузка усиливает структуру; хрестоматийный пример Талеба — именно кость) → адаптивность.
Одна метафора накрывает все три аксиомы. Это и есть искомая квинтэссенция, выраженная образом.

## Часть II. Кандидаты и скрининг

Автопрогон (GitHub-org, crates.io, npm, PyPI, RDAP .dev/.org). 404 = свободно.

### Костная ткань / несущая структура
- **Trabecula / Trabecula OS** — ⭐ лидер. Лат. «маленькая балка»; губчатая кость = адаптивный несущий граф. Скрин: crates **free**, npm **free**, PyPI **free**, .dev/.org **free**, GitHub-org свободен. В tech-поиске чисто — только анатомия/офтальмология (Trabectome — мед. прибор, другой класс), путаницы с софтом нет. Минус: 4 слога, чуть длинно; лечится формой **Traba** или доменом trabecula.dev + CLI `trab`.
- **Osteon** — структурная единица компактной кости (каналы + концентрические пластинки). Короче, звучит твёрже. Скрин: crates/npm/PyPI/.dev **free**. Коллизия: osteon-medical на GitHub (мед-имиджинг) + Osteon India (материалы) — не софт-ОС, но соседство ближе, чем у Trabecula. Твёрдый второй.
- **Trabekula** (через k) — .dev/.org/crates **free**, разводит с лат. написанием, но теряет узнаваемость слова. Запасной.
- Отброшены: tecton (звучит, но .org занят, tectonic — известный TeX-движок), phalanx/tessera (crates заняты).

### Античность (Рим/Греция), отсеки, структура
- **Insula** — рим. «остров»: и городской жилой блок (модуль!), и «остров» (local, автономный). Двойной смысл ложится на модульность + local-first. Скрин: crates/GitHub-org **free**, .dev/.org заняты (домены — не блокер для OSS). Сильный вариант с античной отсылкой.
- **Castrum** — рим. военный лагерь, собираемый из стандартных модулей по типовой сетке (античный «компонентный UI»). Скрин: **всё free** (GitHub-user, crates, .dev, .org). Смысл модульности/структуры, deadpan-твёрдый. Недооценённый кандидат.
- **Limina** (мн. от limen, «порог») — пороги/переходы, красиво ложится на «перетекания» FlexLight. crates/.dev/.org **free**. Минус: смысл тоньше, менее очевиден.
- **Enchiridion** — «руководство под рукой» (Эпиктет); всё free, но 5 слогов — проваливает телефонный тест. Только как кодовое имя.
- Отброшены: agora/forum (crates заняты, затёрты донельзя), cohort (медицина/аналитика).

### Антихрупкость / адаптивность
- **Hormesis** — доза стресса усиливает систему (биологическая основа антихрупкости). crates/.dev **free**, .org занят. Смысл 10/10, но слово почти никто не знает — придётся объяснять всегда. Кандидат «на любителя».
- Kintsugi (золочение трещин), radix, rhizome — crates заняты; rhizome вдобавок затёрт (Deleuze, арт-организации).

### Децентрализация / local-first (прямых имён почти не осталось)
- Radix (крипта), rhizome (арт), holon (Holochain) — все заняты/затёрты. Вывод: в лоб «децентрализацию» называть нечем — заходить через метафору структуры (кость/инсула), которая несёт децентрализацию как следствие. Это и есть польза от базиса: не называем следствие, называем корень.

## Часть III. Рекомендация

Метафора кости — единственная, что покрывает весь минимальный базис одним образом (граф + локальная перестройка + антихрупкость). Поэтому финал — оттуда, с античным резервом:

1. **Trabecula** — семантически точнейшее, в tech чисто по всем реестрам. Основной кандидат. Формы: бренд `Trabecula`, домен `trabecula.dev`, CLI `trab`, короткая форма `Traba` для устного.
2. **Castrum** — если хочется короче/жёстче и с римской отсылкой (модульный лагерь по сетке); всё свободно.
3. **Osteon** — короткий костный вариант; чуть ближе мед-соседство.
4. **Insula** — если ценнее «остров + модуль» (local-first в самом слове), мириться с занятыми доменами.

Семейство подпроектов от Trabecula выводится само: остеобласт-строитель = **Osteo** (агент, наращивающий граф), остеокласт-сборщик = **Clast** (GC/забвение), несущая ось = **Lamella** (слой представления), сигнал нагрузки = **Wolff** (движок перестройки по закону Вольфа). Костный словарь богат и никем в tech не занят.

**Следующий шаг:** неделя обкатки Trabecula + Castrum параллельно (писать в спеках, произносить, «сказать в Xecut»), затем ручной TMview (классы 9/42, EUIPO+WIPO) для финалиста и день захвата. Обкаточный тест: «Мы делаем Trabecula» — если переспрашивают про смысл максимум один раз и он «залипает» после объяснения про кость под нагрузкой — имя живёт.

### Обновление к плейбуку (общий вывод по методу)
Правило, подтверждённое тремя раундами: **не называй следствие — называй корень.** Прямые имена для «fluid», «decentralized», «local-first», «agentic» либо заняты, либо затёрты, потому что все ищут там же. Имя от несущей метафоры (кость) уникально и при этом выводит все следствия само — ровно как базис выводит идеи системы.
