# Naming Playbook: как назвать ОС

> Ревизия: 10.07.2026. Задача: заменить рабочее имя HyperOS (занято Xiaomi, сотни миллионов устройств, трейдмарк). Здесь: метод, критерии, семейства кандидатов, результаты живых проверок трёх финалистов, следующие шаги.

## 0. Диагноз после живых проверок

Проверил три кандидата из разных семейств — все заняты или тесно:

- **Holon** (Кёстлер: целое-и-часть — идеальный смысл для entity-ОС): академическая область «holonic systems», мёртвый Holon Linux на DistroWatch, город в Израиле, и главное — соседство с экосистемой Holochain (HoloPortOS, Holonix). Не смертельно, но вечная путаница.
- **Hypha** (нить мицелия): выжжено — hypha.earth (DAO-платформа), LF Energy Hyphae, PyPI-пакет hypha (AI-фреймворк), HyphaApp (OTF), агентства. Плюс урок: по-русски hypha — «гифа», бренд расщепляется на два слова.
- **Sreda** (среда): компания Sreda Solutions (ирония: embedded/OS-контора из Петербурга), Sreda.ai, телекомпания «Среда». Тесно.

**Вывод, который важнее кандидатов: все обычные существительные в tech уже разобраны.** Работают три стратегии: (а) редкое реальное слово вне tech-употребления (путь Fuchsia), (б) коин из двух морфем, (в) слово из неанглийского языка, читающееся одинаково везде. Отсюда метод.

## 1. Критерии

**Hard-фильтры (не прошёл — вылетел):**
1. Гугл-тест: по «<имя> os» и «<имя> software» на первой странице нет живых проектов; через год после запуска реально быть результатом №1.
2. Свободно (или мертво) на: GitHub orgs, crates.io (мы Rust-first!), npm, PyPI.
3. TMview/WIPO: нет живых знаков в классах 9 и 42 (ЕС + США минимум) на software/OS.
4. Домен: <имя>.dev или <имя>.org достижим (для OSS .com не обязателен).
5. Читается одинаково EN и RU, ≤3 слога, без транслитерационного расщепления (анти-пример: Hypha/гифа).
6. CLI-тест: имя строчными в терминале выглядит и набирается (`fuchsia`, `zircon`): без дефисов, цифр, CamelCase-каламбуров.
7. Телефонный тест: продиктовать имя по телефону без спеллинга.
8. Нет мусорных/негативных значений в топ-10 языках (прогнать через словари/LLM).
9. Не начинается с Hyper/Hypr (Xiaomi + Hyprland) и не заканчивается на «OS» как часть слитного слова рядом с чужими знаками.

**Soft-скоринг (0–2 каждый):** смысловая связь с entity/agent/local-first · расширяемость в семейство имён подпроектов · deadpan-совместимость (не пафос, не «Quantum Nexus AI») · личный отклик (жить с этим годами).

## 2. Процесс (5 шагов)

1. **Генерация**: 40–60 кандидатов по семействам (§3), без цензуры.
2. **Стол**: hard-фильтры 5–9 без интернета → останется 8–12.
3. **Живые проверки**: Google → GitHub → crates.io → TMview → домены. Порядок именно такой: от дешёвой проверки к дорогой. Останется 3–5.
4. **Неделя обкатки**: переименовать локальную ветку спек в кандидата, произносить вслух, тест «сказать в Xecut и посмотреть на лица», показать Denni и Лене. Имя, которое неловко произносить, умирает здесь.
5. **Захват**: GitHub org + домены + заглушки на crates.io в один день; заявка на TM (ЕС через EUIPO — приоритетно, дальше по месту) — можно позже, до публичности.

Инструменты: tmview.euipo.europa.eu · branddb.wipo.int · crates.io · namechk.com · domainr.com.

## 3. Семейства и кандидаты (после стола, до живых проверок)

**A. Мицелий — главный кандидат в семейства.** У тебя уже есть Mycelia (privacy-first память). Если ОС берёт это семейство, портфель складывается сам, а словарь подпроектов бесконечен: имена внутри проекта не требуют уникальности в мире.
- **Mycel** — короче Mycelia, произносится одинаково RU/EN («мицел»), CLI-пригодно. Проверить: mycel.dev, crates.io (был p2p-проект Mycel — проверить жив ли), TM.
- Схема подпроектов: entity store = *Spore*, синк = *Rhizo*, shell = *Cap*, агентный слой = *Hypha* (внутри проекта — можно!), история = *Ring* (годовые кольца).
- Риск: Mycelium — известный биткоин-кошелёк; сам «Mycel» тоже мог быть занят — живая проверка обязательна.

**B. Целое-и-часть (мереология):**
- **Meron** — термин мереологии (часть в отношении «часть-целое»): смысл 10/10 для entity-ОС, редкое, два слога, читается одинаково. Проверить: имя собственное (есть люди/бренды Meron), TM класс 9.
- Holon — вычеркнут (см. §0), но держать как объяснительную метафору в доках: «сущности HyperOS холоничны».

**C. Русско-латинские (уникальность почти гарантирована, deadpan в комплекте):**
- **Ostov** (остов — каркас, скелет): внутри слова живёт «OS», по-английски читается «OSS-tov», уникально в tech. Минус: для не-русских — просто звук.
- **Vetka** (ветка): git-родное слово, тёплое, произносимое. Минус: уменьшительность.
- Strizh (стриж — та самая птица swift; Swift занят Apple, а Strizh свободен — шутка на поверхности) — минус: «zh» ломает телефонный тест на Западе.

**D. Коины из морфем:**
- **Entos** (entity + OS; заодно ἐντός — «внутри» по-гречески, привет local-first): проверить — была химическая AI-компания Entos.
- Noos (νοῦς, ноосфера — Вернадский, русский резонанс): рядом лежал Noosphere-протокол Subconscious (проект закрыт — имя, возможно, освободилось де-факто, но не де-юре).

**E. Запасная стратегия — «путь Fuchsia»:** взять слово, вообще не связанное со смыслом (цвет, минерал, гриб, река), редкое в tech. Смысл нарастает на имя, а не наоборот — Fuchsia это доказала. Кандидатов генерить списками: минералы (не Zircon/Graphene), съедобные грибы с латинскими именами, сибирские реки. Правило то же: hard-фильтры → живые проверки.

## 4. Рекомендация

1. Прогнать по §2 в первую очередь: **Mycel, Meron, Ostov, Entos** + 2–3 из запасной стратегии.
2. Ставка по умолчанию — семейство мицелия: единственный вариант, где имя ОС усиливает уже существующий проект (Mycelia) и даром даёт систему имён подпроектов. Если Mycel занят — искать в том же семействе (лат. названия грибов/структур: Velum, Gleba, Peridium — у каждого проверить смыслы, Gleba по-русски смешно, что может быть и плюсом).
3. До выбора имени — в репозитории и спеках заменить «HyperOS» на нейтральный кодовый плейсхолдер (например `PROJECT_K` или кодовое имя сессии), чтобы имя Xiaomi не вросло в код, доки и URL ещё глубже.
4. День захвата (§2 шаг 5) — сразу после решения, до любого анонса, включая посты в Threads.

## Финалисты

**1. FluidSpace — промоушен собственного имени.** Неожиданный лидер по данным: GitHub-org **свободен**, crates.io **свободен**, fluidspace.dev **свободен** (.org занят). Это твоё же слово из UX RESEARCHES, идеальное попадание в собственный вкус, направление fluid — первое в списке. Смысл масштабируется с виджета на ОС честно: вся система и есть «текучее пространство» (ZUI, stories, перетекания FlexLight). Цена: внутренний концепт FluidSpace переименовать (кандидаты: Sheets, Canvas, Paper — он и описан как «Notion+Miro+Dropbox»). CLI: `fluidspace` длинновато, но сокращение `fs`... занято понятием filesystem — использовать `flsp`/`fluid`.
- Риск TM: слова fluid+space по отдельности слабые, комбинация регистрабельна; проверить TMview класс 9/42.

**2. Fluidum.** Латинское «текучее», путь Fuchsia (редкое реальное слово). По данным лучший из односложных: fluidum.dev **свободен**, crates **свободен**, npm **свободен**. Читается одинаково RU/EN, 3 слога, CLI-идеально. Коллизия одна: fluidum.ee — эстонская AI-консалтинг-студия (класс 42 — проверить TMview EUIPO; географически близко, но услуги ≠ ОС). Бонус-смысл: «флюид» — то, что незримо связывает; эзотерический привкус деадпан-совместим.

**3. Entos** (entity + OS; ἐντός — «внутри», привет local-first). crates и npm **свободны**, entos.dev занят. Коллизии: была химическая AI-компания Entos (переименовалась в Iambic — судьбу знака проверить), мелкие Entos-фирмы. Смысл прямолинейный, произношение идеальное. Твёрдый третий.
