```yaml term-map
entries:

    domain: product
    definition: A Rust mobile OS prototype with Fuchsia-inspired architecture, targeting a Pixel 9 form-factor simulator.
  - term: Pixel 9 form-factor simulator

  - term: capability handle
    domain: architecture
    definition: An explicit transferable authority token used for IPC and resource access instead of ambient global permission.
  - term: spec-led
    domain: workflow
    definition: A repository rule that authoritative specs are written and read before dependent implementation work starts.
  - term: build status
    domain: workflow
    definition: The documented current ability or inability to run build and test commands safely.

```

These terms anchor the bootstrap specs and should be refined in follow-up specs
when subsystem vocabulary becomes precise.
